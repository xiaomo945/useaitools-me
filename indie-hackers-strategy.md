#### 模板 2：技术分享
```
标题：我是如何用 Next.js 和静态 JSON 构建一个 80+ 工具的导航站的（无数据库，$0 托管成本）

正文：
大家好！我想分享一个技术决策：为什么我选择用静态 JSON 文件而不是数据库来存储 80+ 个 AI 工具的数据。

## 背景
UseAITools.me 是一个 AI 工具导航站，包含：
- 80+ 工具详情
- 776+ 博客文章
- 6 个分类
- 多个对比页面

## 为什么不用数据库？

### 1. 成本考虑
- 数据库托管：$15-50/月
- 静态 JSON + Vercel：$0/月

### 2. 性能优势
- 页面加载时间：< 1 秒
- 无需 API 调用
- 完美的 SEO（纯静态 HTML）

### 3. 维护简单
- 不需要数据库迁移
- 不需要备份策略
- Git 就是版本控制

## 具体实现

### 数据结构
```json
{
  "id": 1,
  "name": "Tool Name",
  "description": "...",
  "category": "Writing",
  "pricing": "Free",
  "url": "https://...",
  "affiliate_link": "https://..."
}
```

### 动态路由
使用 Next.js 的 `getStaticPaths` 和 `getStaticProps`：
```javascript
export async function getStaticPaths() {
  const tools = fs.readFileSync('./data/tools.json');
  const paths = tools.map(tool => ({
    params: { id: tool.id.toString() }
  }));
  return { paths, fallback: false };
}
```

### Sitemap 生成
从 JSON 文件动态生成 sitemap.xml，包含所有 80+ 工具页面和 776+ 博客文章。

## 性能数据
- Lighthouse 分数：98/100
- 首次内容绘制：< 1 秒
- 累计布局偏移：0

## 什么时候应该用数据库？
如果你的项目需要：
- 用户生成内容
- 实时数据更新
- 复杂的查询和过滤
- 用户认证和权限

那么数据库是必需的。但对于内容型网站，静态 JSON 是一个被低估的选择。

## 项目链接
- 网站：https://useaitools.me
- GitHub：https://github.com/xiaomo945/useaitools-me

欢迎提问和交流！
```

## 📊 Karma 获取策略

### 高 Karma 行为
- ✅ 回答未解决的问题（+10-50 Karma）
- ✅ 分享详细的经验帖（+20-100 Karma）
- ✅ 提供有用的代码示例（+5-20 Karma）
- ✅ 分享免费资源/工具（+10-30 Karma）
- ✅ 帮助新手解答基础问题（+5-15 Karma）

### 避免的行为
- ❌ 只发链接不提供价值（-5-20 Karma）
- ❌ 过度自我推广（-10-50 Karma）
- ❌ 参与争议性讨论（-5-30 Karma）
- ❌ 发布低质量内容（-5-15 Karma）

## ⚠️ 注意事项

### 社区规则
- 仔细阅读 Indie Hackers 的社区规则
- 不要违反自我推广政策（通常要求 9:1 的比例：9 个非推广内容 : 1 个推广内容）
- 尊重他人，避免争议性话题

### 内容质量
- 每条回复至少 3-5 句话
- 提供具体例子和数据
- 避免泛泛而谈
- 使用格式化（列表、粗体、代码块）提高可读性

### 时间管理
- 每天固定时间浏览和回复（建议 30 分钟）
- 不要一次性发太多评论（会被视为刷 Karma）
- 优先回复有深度的问题
