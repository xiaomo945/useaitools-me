## ⚙️ 工作原则

### 1. 测试完立即推送
**规则**: 每次完成任务并测试通过后，立即推送到主分支，避免积累大量未推送的更改。

**执行流程**:
1. 完成任务
2. 运行测试: `npm run build`
3. 确认无错误
4. 提交更改: `git add . && git commit -m "完成: [任务描述]"`
5. 推送到主分支: `git push origin main`

### 2. 代码层级优先
**规则**: 优先执行代码层级的优化，需要人工干预的任务记录后延后。

**优先级**:
1. 🔴 数据库基础设施（必须）
2. 🟠 用户系统（必须）
3. 🟡 内容优化（重要）
4. 🟢 SEO 优化（重要）
5. 🔵 用户体验（锦上添花）

### 3. 一次一个任务
**规则**: 每次只执行一个任务，完成后询问用户是否继续。

**执行流程**:
1. 执行当前任务
2. 测试验证
3. 推送到主分支
4. 询问用户: "任务 X 已完成，是否继续任务 X+1？"

### 4. 不要骗自己
**规则**: 诚实评估进度，不要夸大成果。

**评估标准**:
- 功能是否真正可用？
- 数据是否真正持久化？
- 用户是否真正能用？
- SEO 是否真正有效？
