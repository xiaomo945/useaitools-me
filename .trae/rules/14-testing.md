## 🔮 核心哲学
1. **测试即文档**：好的测试用例比任何文档都更清晰地描述代码的行为。
2. **自动化即信心**：每一次自动化测试通过，都是对代码质量的一次投票。
3. **边界即重点**：Bug 最爱藏在边界条件里。空数组，空字符串、null、undefined 都是测试重点。
4. **覆盖率即底线**：核心逻辑必须有 100% 的测试覆盖。
5. **快速即价值**：测试必须在 5 分钟内跑完，否则开发者会跳过它。

## 📐 实施规范

### 测试框架选型
- **单元测试**：Vitest（与 Vite 生态完美兼容，速度快）
- **组件测试**：React Testing Library + Vitest
- **E2E 测试**：Playwright（支持多浏览器、视觉回归）
- **视觉回归**：Chromatic 或 Percy（可选，后期添加）

### 核心单元测试
- **搜索过滤函数**：测试空搜索、单字符搜索、特殊字符搜索，大写不敏感
- **分类过滤函数**：测试全部/单个分类/不存在的分类
- **排序函数**：测试按名称/按分类/按价格的排序结果
- **收藏逻辑**：测试添加/移除/重复添加/清空收藏
- **环境变量读取**：测试正常读取/缺失回退/格式错误

### 核心组件测试
- **工具卡片**：测试渲染、价格标签、分类标签、CTA按钮点击
- **搜索框**：测试输入、清除、防抖
- **分类按钮组**：测试所有按钮渲染、点击切换、高亮状态
- **工具对比页**：测试选择器、对比表格，空状态

### 核心 E2E 用户路径
- **搜索路径**：首页 → 搜索"Rytr" → 看到Rytr卡片 → 点击Visit Website → 跳转正确
- **分类路径**：首页 → 点击"Writing" → 只显示Writing工具 → 点击工具卡片 → 进入详情页
- **对比路径**：首页 → 点击"Compare" → 选择两个工具 → 看到对比表格
- **收藏路径**：首页 → 点击Save → Saved Tools数量+1 → 进入Saved页面 → 看到已收藏工具

### 测试覆盖率目标
- **核心工具函数**：100%
- **关键组件**：≥80%
- **E2E 核心路径**：100%
- **视觉回归**：关键页面（首页、详情页、分类页）

### CI 质量门禁
- **Lint 检查**：`eslint` 无错误
- **类型检查**：`tsc --noEmit` 无错误
- **单元测试**：`vitest run` 全部通过
- **构建检查**：`npm run build` 无错误
- **E2E 测试**：`playwright test` 关键路径通过
