# 🚀 性能优化行动指南

**创建时间**: 2026-06-15  
**负责团队**: Frontend Developer + Performance Benchmarker + SRE  
**目标**: 首页加载时间从 5s+ 降至 2.5s 以内（LCP ≤ 2.5s）

---

## 📊 当前性能基线

| 指标 | 当前值 | 目标值 | 状态 |
|------|--------|--------|------|
| LCP (Largest Contentful Paint) | ~5s+ | ≤ 2.5s | ❌ 不达标 |
| FID (First Input Delay) | 待测量 | ≤ 100ms | ⚠️ 待确认 |
| CLS (Cumulative Layout Shift) | 待测量 | ≤ 0.1 | ⚠️ 待确认 |
| 首页数据加载量 | ~1358个工具 | 12个工具 | ❌ 过大 |
| 初始Bundle大小 | 待测量 | < 200KB | ⚠️ 待确认 |

## 🎯 优化任务清单

### P0: 立即执行（影响最大）

#### 任务 1.1: 数据加载优化 ⚡
**优先级**: P0  
**负责**: Frontend Developer  
**预计影响**: LCP 减少 2-3s  
**完成度**: 0%

**问题**: 首页从tools.json加载所有1358个工具数据，但只显示12个

**解决方案**:
1. 创建API路由 `/api/tools` 支持分页
2. 首页只加载前12个工具
3. 实现"加载更多"功能，按需获取数据
4. 添加加载状态和错误处理

**验收标准**:
- [ ] 首页初始加载只包含12个工具数据
- [ ] "加载更多"按钮可正常加载剩余工具
- [ ] 每次加载12个工具，直到全部加载完成
- [ ] 加载过程有骨架屏或加载指示器
- [ ] 构建验证通过

**实施步骤**:
```
1. 创建 /app/api/tools/route.ts
2. 实现分页逻辑（page, limit参数）
3. 修改 page.tsx，只获取前12个工具
4. 修改 HomeClient.tsx，实现"加载更多"
5. 添加加载状态UI
6. 测试验证
```

#### 任务 1.2: 第三方脚本优化 ⚡
**优先级**: P0  
**负责**: Frontend Developer  
**预计影响**: LCP 减少 0.5-1s  
**完成度**: 0%

**问题**: Clarity和Analytics脚本在<head>中同步加载，阻塞渲染

**解决方案**:
1. 使用 `next/script` 组件
2. 设置 `strategy="lazyOnload"` 延迟加载
3. 在用户首次交互后加载分析脚本

**验收标准**:
- [ ] Clarity脚本使用lazyOnload策略
- [ ] Analytics脚本使用lazyOnload策略
- [ ] 脚本在用户交互后加载
- [ ] 构建验证通过
- [ ] 分析数据正常上报

**实施步骤**:
```
1. 修改 layout.tsx
2. 将 <script> 标签替换为 <Script> 组件
3. 添加 strategy="lazyOnload"
4. 测试验证脚本加载时机
```

- [ ] 构建验证通过

**实施步骤**:

#### 任务 1.4: 组件懒加载优化
**优先级**: P1  
**负责**: Frontend Developer  
**预计影响**: 初始Bundle减少 30-40%  
**完成度**: 0%

**问题**: HomeClient组件过大，包含所有交互逻辑

**解决方案**:
1. 将HomeClient拆分为多个小组件
   - SearchBar（独立组件）
   - CategoryFilters（独立组件）
   - ToolGrid（独立组件）
   - LoadMoreButton（独立组件）
2. 使用 `dynamic()` 懒加载非首屏组件
3. 使用 `React.memo()` 优化重渲染

**验收标准**:
- [ ] HomeClient拆分为4个独立组件
- [ ] 非首屏组件使用dynamic()懒加载
- [ ] 首屏组件使用React.memo()优化
- [ ] 初始Bundle大小 < 200KB
- [ ] 构建验证通过
- [ ] 功能测试通过

**实施步骤**:
```
1. 创建 SearchBar.tsx 组件
2. 创建 CategoryFilters.tsx 组件
3. 创建 ToolGrid.tsx 组件
4. 创建 LoadMoreButton.tsx 组件
5. 修改 HomeClient.tsx，导入新组件
6. 使用dynamic()懒加载非首屏组件
7. 测试验证
```
