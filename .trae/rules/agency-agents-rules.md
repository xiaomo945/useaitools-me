## 🤖 自动引用规则

### 规则 1: 任务-专家映射

当用户提出任务时，Trae 应自动识别并引用相关专家：

**示例 1**: "优化首页加载速度"
- 引用: `engineering-frontend-developer.md` + `testing-performance-benchmarker.md`
- 动作: 分析性能瓶颈，实施优化方案

**示例 2**: "写一篇关于 ChatGPT 的评测文章"
- 引用: `marketing-content-creator.md` + `marketing-seo-specialist.md`
- 动作: 撰写高质量内容，优化 SEO 元数据

**示例 3**: "设计新的工具卡片交互"
- 引用: `design-ui-designer.md` + `design-ux-architect.md`
- 动作: 设计交互方案，确保用户体验一致性

**示例 4**: "分析上个月的流量数据"
- 引用: `support-analytics-reporter.md` + `marketing-growth-hacker.md`
- 动作: 生成数据报告，提出增长建议

**示例 5**: "检查网站是否符合 WCAG 标准"
- 引用: `testing-accessibility-auditor.md` + `design-inclusive-visuals-specialist.md`
- 动作: 执行无障碍审计，修复合规问题

### 规则 2: 多专家协作

复杂任务需要多个专家协作：

**示例**: "提升联盟营销转化率"
1. **Business Strategist** - 分析商业模式，优化变现策略
2. **Growth Hacker** - 设计 A/B 测试，优化 CTA 按钮
3. **UI Designer** - 改进视觉设计，提升点击率
4. **Analytics Reporter** - 追踪转化数据，生成报告
5. **Pricing Analyst** - 优化联盟佣金结构

### 规则 3: 阶段化执行

大型项目按阶段执行，每个阶段调用对应策略专家：

**示例**: "从零到一构建 AI 工具导航站"
1. **Phase 0: Discovery** - 市场调研、竞品分析、需求定义
2. **Phase 1: Strategy** - 技术选型、架构设计、路线图
3. **Phase 2: Foundation** - 数据库搭建、用户系统、基础功能
4. **Phase 3: Build** - 核心功能开发、内容创作、SEO 优化
5. **Phase 4: Hardening** - 性能优化、安全加固、测试覆盖
6. **Phase 5: Launch** - 产品发布、社交媒体推广、用户获取
7. **Phase 6: Operate** - 数据监控、持续迭代、规模化增长

## 📝 使用指南

### 如何调用专家

**方式 1: 直接引用**
```
请阅读 agency-agents/marketing/marketing-seo-specialist.md，然后优化首页的 SEO 元数据。
```

**方式 2: 描述任务**
```
我需要写一篇关于 Midjourney 的深度评测，要求：
- 2000 字以上
- 包含优缺点分析
- 优化 SEO 关键词
- 符合品牌语调
```
Trae 会自动引用: `marketing-content-creator.md` + `marketing-seo-specialist.md` + `design-brand-guardian.md`

**方式 3: 阶段化执行**
```
我们准备启动"国际化"项目，请按阶段执行：
1. 先做市场调研（Phase 0）
2. 制定战略（Phase 1）
3. 搭建基础（Phase 2）
```
Trae 会依次引用: `phase-0-discovery.md` → `phase-1-strategy.md` → `phase-2-foundation.md`
